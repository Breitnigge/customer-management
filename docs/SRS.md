# SRS: 顧客管理システム

## ドキュメント管理
- バージョン: 1.0
- 作成日: 2026-07-11
- ステータス: Approved

---

## 1. はじめに

### 1.1 目的
個人事業主が顧客の名前・連絡先・購入履歴を一元管理するWebアプリを提供する。

### 1.2 スコープ（IN/OUT）

**IN SCOPE**
- 顧客の登録・閲覧・編集・削除
- 購入履歴の登録・閲覧
- 顧客名・連絡先でのインクリメンタル検索
- DBファイルバックアップダウンロード

**OUT OF SCOPE**
- マルチユーザー・認証
- メール送信・外部API
- CSV/グラフ機能

## 3. 機能要件

### FR-CUST-001: 顧客登録
- **要件**: 名前（必須）・電話番号・メールアドレス・メモを入力して顧客を登録できる
- **優先度**: Must
- **Evidence ID**: EV-001（SQLiteの永続化による）
- **受入基準**: ACC-FR-CUST-001 — 必須項目の名前を入力してSubmitすると、一覧に表示される
- **検証方法**: TC-FR-CUST-001
- **例外条件**: 名前が空の場合はバリデーションエラーを表示

### FR-CUST-002: 顧客一覧表示
- **要件**: 登録済みの顧客を一覧で表示する
- **優先度**: Must
- **Evidence ID**: EV-001
- **受入基準**: ACC-FR-CUST-002 — 登録した顧客が一覧テーブルに表示される
- **検証方法**: TC-FR-CUST-002
- **例外条件**: 0件の場合は「顧客が登録されていません」と表示

### FR-CUST-003: 顧客検索
- **要件**: 名前または連絡先（電話/メール）でインクリメンタル検索できる
- **優先度**: Must
- **Evidence ID**: EV-001
- **受入基準**: ACC-FR-CUST-003 — 検索ボックスに入力すると一覧がリアルタイムで絞り込まれる
- **検証方法**: TC-FR-CUST-003
- **例外条件**: 0件ヒットの場合は「該当する顧客が見つかりません」と表示

### FR-CUST-004: 顧客詳細・編集
- **要件**: 顧客をクリックして詳細を閲覧し、情報を編集できる
- **優先度**: Must
- **Evidence ID**: EV-001
- **受入基準**: ACC-FR-CUST-004 — 編集してSaveすると更新内容が保存・反映される
- **検証方法**: TC-FR-CUST-004
- **例外条件**: 名前を空にしての保存はエラー

### FR-CUST-005: 顧客削除
- **要件**: 顧客を削除できる（確認ダイアログあり）
- **優先度**: Must
- **Evidence ID**: EV-001
- **受入基準**: ACC-FR-CUST-005 — 確認後に削除すると一覧から消える
- **検証方法**: TC-FR-CUST-005
- **例外条件**: キャンセルで削除中止

### FR-HIST-001: 購入履歴登録・表示
- **要件**: 顧客詳細画面で購入日・商品名・金額を登録でき、一覧で閲覧できる
- **優先度**: Must
- **Evidence ID**: EV-001
- **受入基準**: ACC-FR-HIST-001 — 購入履歴を登録すると顧客詳細の履歴欄に表示される
- **検証方法**: TC-FR-HIST-001
- **例外条件**: 商品名が空の場合はエラー

### FR-BACKUP-001: DBバックアップダウンロード
- **要件**: `/api/backup` でSQLiteファイルをダウンロードできる
- **優先度**: Must（データ消失防止の中核）
- **Evidence ID**: EV-001
- **受入基準**: ACC-FR-BACKUP-001 — エンドポイントにアクセスするとDBファイルがダウンロードされる
- **検証方法**: TC-FR-BACKUP-001
- **例外条件**: ファイルが存在しない場合は500エラー

## 4. 非機能要件

### NFR-PERF-001: 検索レスポンス
- 検索クエリのAPIレスポンスは 500ms 以内

### NFR-SEC-001: データ保護
- DBファイルはサーバーローカルに保存（外部送信なし）

### NFR-AVAIL-001: 可用性
- ローカル起動（シングルユーザー）のため99%稼働は不要。起動できること。
