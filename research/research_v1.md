# リサーチ V1

## 観点①: ツール/フレームワーク

| 候補 | 信頼度 | 採否 | 理由 |
|---|---|---|---|
| Next.js 14 (App Router) | A | 採用 | フロント+APIを1リポジトリで管理。シングルユーザー用途に最適。ドキュメント豊富 |
| Express.js + 別フロント | A | 不採用 | リポジトリ分割が不要な複雑さ。今回のスコープに対してオーバーエンジニアリング |
| Remix | B | 不採用 | Next.js と用途が重複し学習コストが高い |

## 観点②: データベース

| 候補 | 信頼度 | 採否 | 理由 |
|---|---|---|---|
| SQLite + better-sqlite3 | A | 採用 | ファイルベースでサーバー不要。バックアップ=ファイルコピーで完結。データ消失リスク最低 |
| PostgreSQL / MySQL | A | 不採用 | サーバー管理が必要。シングルユーザー用途に対して過剰 |
| LocalStorage/JSON ファイル | B | 不採用 | 検索・結合クエリが書けない |

## 観点③: UIスタイリング

| 候補 | 信頼度 | 採否 | 理由 |
|---|---|---|---|
| Tailwind CSS | A | 採用 | Next.js と親和性高。ユーティリティクラスで高速UI実装 |
| shadcn/ui | A | 採用（補助） | Tailwindベースのコンポーネントライブラリ。テーブル・フォームを再利用 |

## 確定技術スタック

```
Next.js 14 (App Router)
  ├── better-sqlite3 (DB操作)
  ├── Tailwind CSS (スタイリング)
  └── データ: data/customers.db (SQLiteファイル)
```

## データ消失リスク対策（EV-001）

- SQLite はファイル1つ → Time Machine / 手動コピーでバックアップ可
- `/api/backup` エンドポイントでDBファイルをダウンロード可能にする
- ソースコード: Git管理推奨
